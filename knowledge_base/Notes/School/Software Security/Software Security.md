Develop an adversarial mindset: if I were outside and I needed to get in, how could I get in

Software doesn’t run in isolation. It depends on users to operate it, configuration, an OS, a Hypervisor, etc.

Exploits leverage vulnerabilities

What assumptions is the system making?
Who processes the data?
What buttons can I press to break this?

## Ethics

Never hack or attempt to find vulnerabilities in systems you don’t own or have permission to do so

Download source code
Participate in bug bounty
Become an academic

Options after discovering a vulnerability:
- Sell the information
- Tell the company
- Tell the world

Ethical hacking: pentesting
- Source code analysis
- Social engineering
- Policy analysis
- etc
